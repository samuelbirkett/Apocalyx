Introduction

This document describes the use of the LUA language (and eventually the included Tiny C compiler, AngelScript interpreter, SMALL interpreter, CSL interpreter or FASM assembler) to create scripts for the APOCALYX 3D Engine.

The Lua language is a powerful scripting language easily extendable and embeddable in other applications. Here the description of LUA is limited to the features that are useful to create APOCALYX scripts, but a lot of other possibilities are available. To know more about LUA visit http://www.lua.org

The APOCALYX 3D Engine is an OpenGL based 3D engine with interesting features and powerful scripting capabilities. The use of LUA makes possible to create applications accessing easily all the exposed API of the engine. To know more about APOCALYX visit http://apocalyx.sf.net

APOCALYX includes a console from which you can execute LUA scripts. To show the console, press the 'F3' key. Using the command line of the console, you can execute interactively single or multiple lines written in the LUA language, run LUA scripts or compile them.

The available commands are:

  • h : to read the complete list of the commands
  • l : to read a list of LUA scripts
  • r "file name" : to execute a LUA script
  • c "file name" : to compile a LUA script (the output is written in 'luac.out'). Remember that LUA scripts don't need to be compiled before execution: Compilation is a plus to speed up the loading phase
  • i : to enter and interpret multiple LUA lines
  • i "string" : to interpret a single LUA string
  • d : to change the working directory
  • e "file name" : to edit text files
  • g "file name" : to execute external programs
When a script is in execution, you can read all the messages that the script generates on the console.

To execute a particular script you can also specify its name on the command line. For example, the command APOCALYX script.lua executes the script.lua script, while APOCALYX package.dat tells the application to look for the script main.lua in the zipped file package.dat. In no file is specified, the program looks by default for the script main.lua or the file main.dat in the same folder of the application.
An alternate way to execute the script.lua script is to use the option -dofile"script.lua". This command line option has also another particular use explained below.

Options that one can specify on the command line:

  • -nosetup
    to avoid the opening of the setup dialog at the beginning and read the needed data directly from the file setup.ini.
  • -setup"file_name"
    this option specifies the file that contains the setup parameters. By default, that file is setup.ini. Don't forget the " delimiters that surround the file name (useful in particular when the file name contains space characters).
  • -xpos_x -ypos_y
    these options specify the initial top left corner position of the window.
  • -wwidth -yheight
    these options specify the initial dimensions of the window (in this case data specified in setup.ini is ignored).
  • -nobar
    this option specifies that the window must be created without a title bar.
  • -freemouse
    this option requires that the mouse is not acquired by the engine at the beginning.

Options particularly useful when an instance of the engine is already running (in this case any script or package specified on the command line is ignored):

  • -ontop
    to keep permanently on top an instance of the engine already running.
  • -notop
    to cancel the effect of the previous option.
  • -totop
    to bring to top an instance of the engine already running. Useful in combination with the next options.
  • -dostring"string"
    this option specifies a LUA string that must be interpreted. When another instance of the engine is running, the string is passed to that one and this instance is terminated. Don't forget the " delimiters that surround the string.
  • -dofile"file_name"
    this option specifies a LUA file that must be interpreted. When another instance of the engine is running, the file is passed to that one and this instance is terminated. Don't forget the " delimiters that surround the file name (useful in particular when the file name contains space characters). This option is ignored if -dostring is specified.
