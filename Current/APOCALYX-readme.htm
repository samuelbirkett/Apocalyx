1 - What is APOCALYX?
---------------------

The "APOCALYX 3D Engine" is an OpenGL based 3D engine with
interesting features and powerful scripting capabilities.

The manual of the engine is accessible from the file:
APOCALYX-manual-index.htm.

To get scripted demos of the engine visit the Downloads page
at http://apocalyx.sourceforge.net and get the latest executable
(this one could not be the latest) and one or more demos packages.
Then unzip all the downloaded files in the same folder and run
the "APOCALYX.exe" file. 
If you want to modify the demos, remember that the ".dat" files
are ZIP files the extension of which was renamed (they contain
the media files needed by the demos) and the ".lua" files are
text files (they include the code of the scripts), so you can
use your preferred editor and an unzip utility.
Since several languages are supported, I recommend the use of the
Scintilla Text Editor available at http://www.scintilla.org
It provides sintax highlighting for a lot of different languages
and features useful to manage several scripts at once.

The main scripting language supported by APOCALYX is LUA.
LUA is an embeddable and extendable scripting language.
To know more about this language visit http://www.lua.org
You don't need to know all the features of LUA to write programs
for APOCALYX, so you can simply read the APOCALYX documentation
provided at http://apocalyx.sourceforge.net including a "getting
started" guide on LUA (5.1) and the list of the APOCALYX APIs.

Tested on:

OPSYS        - Windows Vista
CPU          - Core2Duo T7300
RAM          - 2048 Mb
GRAPHIC CARD - GeForce 8600M GT 256 Mb
SOUND CARD   - Realtek High Definition Audio

OPSYS        - Windows XP
CPU          - Athlon 64 3700+
RAM          - 1024 Mb
GRAPHIC CARD - GeForce 6200 256 Mb
SOUND CARD   - Realtek AC'97 Audio

OPSYS        - Windows 98
CPU          - Athlon 800 MHz
RAM          - 256 Mb
GRAPHIC CARD - GeForce 256 32 Mb
SOUND CARD   - SoundBlaster Live! 1024

To test the application you need a PC with Windows 98 or higher
and OpenGL installed. Hardware 3D acceleration is required and
the resolution of the graphic card must use at least 16 bits
mode to run the application fullscreen or in a window.

To recompile the application you need the free C++ Borland
compiler and the Code::Blocks IDE, but the use of C++ sources
is provided only for those power-users who want to modify the
inner workings of the engine.
Normal users can program their demos and games using scripts
and no compiler is necessary because APOCALYX already includes
all the tools to interpret/compile scripts.
Anyway the Borland C++ command line compiler is available for
free and so the Code::Blocks IDE.
For more details about the Borland C++ free compiler, visit
http://www.borland.com/bcppbuilder/freecompiler/
For more details about the Code::Blocks IDE, visit
http://www.codeblocks.org

You must have installed the latest OpenGL drivers for your
graphic card on your machine, then no other tool is required
because all the DLLs and libraries are included in the
development package.

5 - How to compile
------------------

If you possess the Borland C++ 5.51 Command Line Compiler
and the Code::Blocks IDE you can recompile the engine using
the project file APOCALYX.cbp included in the dev/src folder.

Remember to set the include path for the resource compiler
in Code::Blocks or the project will not compile.
You can find that setting in the "Settings" menu, then click
on the "Compiler" item, then on the "Directories" tab. Now
copy the path that you can read under the "Compiler" tab
and, finally, click on the "Resource compiler" tab. You
must paste there the same include path already used
for the compiler.

Remember also that normal users can program their demos
and games with scripts and no compiler is necessary
because APOCALYX already includes all the tools
to interpret/compile scripts.

For more details about the Borland C++ free compiler, visit
http://www.borland.com/bcppbuilder/freecompiler/
For more details about the Code::Blocks IDE, visit
http://www.codeblocks.org


6 - Known Bugs
--------------

As a rule, you can fully trust only features that
appear in one of the demos provided with the engine.
Report any bug that you may find using other functions.

If you experience severe malfunctions (system crashes),
they are usually caused by old OpenGL drivers, thus
visit the site of your graphic card manufacturer to
download the latest drivers.
Very old configurations are supported (for example,
GeForce with 32 Mb RAM on 800 MHz Athlon) but
executing some demos, tuned for a more performant
hardware, you may experience poor frame rates.

The main bug that made the engine crash in the past
on some configurations was not my fault, because it
was caused by OpenAL. Now the implementation shipped
with the engine is one of the latest from Creative
that works even on my old computer with a clean
Win98 installation and 1999 original drivers.
If you experience problems that you think are
caused by this version of OpenAL, I can provide
an older version that proved to be stable on
obsolete hardware but contains some minor bugs.
